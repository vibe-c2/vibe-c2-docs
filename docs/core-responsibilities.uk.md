# Основні обов'язки

`vibe-c2-core` — це серце площини управління системи.

Це єдиний компонент, який володіє та використовує ключі розшифрування для payload-ів імплантів/сесій.

## Головні обов'язки

## 1) Володіння ключами та криптографічна межа

- Володіти ключовим матеріалом для шифрування імплантів/сесій на рівні застосунку.
- Визначати контекст ключа за допомогою канонічного маршрутного `id`.
- Розшифровувати/перевіряти вхідні зашифровані payload-и.
- Шифрувати/підписувати вихідні payload-и.
- Тримати канали "сліпими" до відкритого тексту (канали ніколи не розшифровують payload-и).

## 2) Оркестрація протоколу та повідомлень

- Приймати канонічні запити каналів (`inbound.agent_message`).
- Валідувати конверт повідомлення та версію схеми.
- Маршрутизувати payload-и у внутрішній конвеєр обробки.
- Повертати канонічний вихідний конверт (`outbound.agent_message`).

## 3) Оркестрація задач та сесій

- Керувати життєвим циклом та станом сесій/імплантів.
- Визначати, які вихідні задачі очікують для кожного `id`.
- Співвідносити вхідні результати з контекстом задачі/сесії.
- Зберігати історію задач та стан виконання.

## 4) Координація модулів

- Координувати модулі каналів, модулі постачальників імплантів та модулі-транслятори.
- Визначати та застосовувати внутрішні контракти між модулями.
- Надавати API площини управління для керування модулями.

## 5) Цілісність даних та аудиту

- Зберігати канонічні події, записи задач та журнали аудиту.
- Забезпечувати відстежуваність через контекст `message_id` / `correlation`.
- Фіксувати чутливі дії та зміни, що впливають на профілі/ключі.

## 6) Політики та контроль доступу

- Застосовувати RBAC/авторизацію для дій оператора.
- Виконувати перевірку політик перед шляхами виконання задач.
- Підтримувати безпекову позицію на межі площини управління.

## 7) Надійність та відновлення

- Забезпечувати ідемпотентність вхідної/вихідної обробки.
- Безпечно обробляти повторні спроби/захист від повторного відтворення.
- Підтримувати консистентність між перезапусками.

## Поза межами обов'язків ядра

- Транспортно-специфічна механіка доставки (належить модулям каналів).
- Деталі рендерингу/парсингу обфускації у транспортних payload-ах (належить модулям каналів, із використанням спільних SDK).
- Особливості транспортного протоколу імплантів, що виходять за межі канонічних контрактів.

## Основний принцип

`implant/session <-> core` — це справжня протокольна розмова.
Канали — це транспортні ретранслятори.
