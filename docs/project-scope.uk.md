# Обсяг проєкту

## Бачення

Побудувати надійну, зручну для оператора C2-платформу з потужною спостережністю та засобами безпеки.

Цей проєкт орієнтований на спільноту.
Його основна мета — дати кожному можливість створювати модулі у стилі vibe-coding та розгортати власну інфраструктуру у стилі vibe-deploy.

## У межах обсягу (Фаза 1)

- Основні можливості сервера (автентифікація, оркестрація, політики).
- Модульна сервісна модель із незалежно розгортаними модулями.
- Міжсервісні канали комунікації на базі RabbitMQ.
- Автентифікація операторів та рольовий доступ.
- Захищений канал зв'язку агент-сервер.
- Базове призначення задач, збір результатів та журналювання аудиту.
- Оркестрація Docker Compose для локального/dev та MVP-розгортання.
- Технічне планування за принципом "документація насамперед".

## Поза межами обсягу (Фаза 1)

- Повноцінний маркетплейс плагінів.
- Просунуті наступальні модулі, що виходять за межі базових робочих процесів тестування.
- Мультитенантна архітектура корпоративного рівня.

## Зацікавлені сторони

- Власник продукту
- Інженерія безпеки
- Backend/платформна інженерія
- Користувачі-оператори

## Критерії успіху

- Критичні робочі процеси задокументовані та узгоджені.
- Технічні вимоги перевірені та знаходяться під контролем версій.
- Базова архітектура підтримує запланований обсяг MVP.
