# Документація Vibe C2

Цей репозиторій містить базову документацію для проєкту **Vibe C2**.

## Призначення

Використовуйте цей сайт документації, щоб:

- Визначати продуктові та технічні вимоги на ранніх етапах.
- Фіксувати архітектурні рішення до початку реалізації.
- Відстежувати обсяг проєкту та обмеження.
- Підтримувати орієнтацію на спільноту та зручність для контриб'юторів.

## Місія спільноти

Проєкт створений для того, щоб люди могли:

- vibe-кодити власні модулі
- vibe-деплоїти власну C2-інфраструктуру
- ділитися та вдосконалювати патерни разом зі спільнотою

## Перші віхи

1. Фіналізувати модульний обсяг проєкту (core-сервер + модулі).
2. Затвердити технічні вимоги (включно з RabbitMQ-меседжингом та Compose-деплоєм).
3. Встановити архітектурну базу та процес ADR.

## Структура

- **Foundations** — обсяг проєкту, технічні вимоги, архітектурний драфт, основні відповідальності, таксономія модулів, потік повідомлень повної системи.
- **Channels** — потік повідомлень каналу, контракти повідомлень, профілі обфускації (концепції, YAML-довідник, приклади), документація модулів каналів та 15-хвилинний гайд зі створення каналу.
- **Go Packages Ecosystem** — документація спільних Go-пакетів: контракти повідомлень протоколу та SDK середовища виконання каналів.
- **ADR** — записи архітектурних рішень для ключових технічних виборів.
