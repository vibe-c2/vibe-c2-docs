# vibe-c2-http-channel

Репозиторій: `https://github.com/vibe-c2/vibe-c2-http-channel`

## Встановлення

```bash
go get github.com/vibe-c2/vibe-c2-http-channel@v0.5.0
```

## Призначення

- перший production-ready HTTP-модуль каналу для Vibe C2
- зчитує конфігурацію рантайму зі змінних середовища (з фолбеком на `.env`)
- завантажує профілі обфускації з відстежуваної теки
- визначає профіль обфускації (`hint` -> перебір увімкнених профілів)
- підтримує об'єктні ланцюги трансформацій

## Конфігурація рантайму

Змінні середовища:

- `CHANNEL_ID` (за замовчуванням: `http-main`)
- `LISTEN_ADDR` (за замовчуванням: `:8080`)
- `C2_SYNC_BASE_URL` (за замовчуванням: `http://localhost:9000`)
- `PROFILES_DIR` (за замовчуванням: `profiles`)

Запуск з фолбеком на файл `.env`:

```bash
go run ./cmd/http-channel --config .env
```

## Поведінка теки профілів

При запуску:

- створює `PROFILES_DIR`, якщо відсутня
- забезпечує існування `default.yaml`
- якщо відсутній, копіює з `examples/profiles/default.yaml`

У рантаймі:

- безперервно відстежує `PROFILES_DIR`
- новостворені файли профілів завантажуються миттєво
- оновлені файли профілів перезавантажуються миттєво
- видалені файли профілів вивантажуються миттєво

## Поведінка профілю за замовчуванням

Профіль за замовчуванням (`default.yaml`) зіставляє `id` та `encrypted_data` з полями тіла з трансформацією base64.

## Інтеграційне тестування

`vibe-c2-http-channel` включає інтеграційні тести з вбудованим симулятором `test-c2-core` для `/api/channel/sync`.

Покриті сценарії (на основі прикладів з `examples/profiles/*.yaml`):

- зіставлення `body`
- зіставлення `header`
- зіставлення `query`
- зіставлення `cookie`
- вибір профілю за підказкою
- конвеєр трансформацій (декодування `base64` на вході + кодування на виході)
- відхилення неоднозначного набору профілів
- профіль за замовчуванням (зіставлення `id` + `encrypted_data` через тіло)

Запуск:

```bash
go test ./...
```

## Примітки

- Цей канал навмисно не має бар'єрів для комунікації імплант -> канал.
- Межі безпеки очікуються на рівні транспорту/інфраструктури поза цим модулем.
