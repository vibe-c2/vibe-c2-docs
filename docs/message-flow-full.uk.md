# Потік повідомлень (повна система)

Ця сторінка документує наскрізний потік через усі основні компоненти:

- Імплант/Сесія
- Модуль каналу
- Основний сервер
- Модуль-транслятор
- Модуль постачальника імплантів

## Наскрізна послідовність

```mermaid
sequenceDiagram
    autonumber
    participant I as Імплант/Сесія
    participant CH as Модуль каналу
    participant CS as Основний сервер (Власник ключів)
    participant TR as Модуль-транслятор
    participant IP as Постачальник імплантів

    I->>CH: id + encrypted_data (beacon/result/check-in)
    CH->>CS: POST /api/channel/sync (inbound.agent_message)
    CS->>CS: Визначити контекст/ключ за id
    CS->>CS: Розшифрувати + перевірити payload
    CS->>TR: Нормалізувати payload
    TR->>IP: Маппінг на схему постачальника
    IP-->>TR: Розпарсена подія
    TR-->>CS: Нормалізована подія C2
    CS->>CS: Зберегти/аудит/оновити стан

    CS->>TR: Сформувати вихідний payload для id (якщо є)
    TR->>IP: Конвертувати у формат імпланта
    IP-->>TR: Payload відкритого тексту постачальника
    TR-->>CS: Нормалізований вихідний відкритий текст
    CS->>CS: Зашифрувати вихідний payload (або no-op конверт)
    CS-->>CH: HTTP 200 outbound.agent_message (encrypted_data)
    CH-->>I: Повернути зашифрований payload відповіді
```

## Примітки

- `імплант/сесія ↔ ядро C2` — це логічна протокольна розмова.
- Канал є транспортним ретранслятором і залишається "сліпим" до відкритого тексту.
- Транслятор та Постачальник імплантів є внутрішніми шарами обробки на стороні ядра.
